================================================================================
                              IMPORTANT
================================================================================

âŒ NEVER USE ROOT ACCOUNT FOR TERRAFORM/DEPLOYMENTS

Why?
- Root has unlimited access (security risk)
- No way to limit what root can do
- If credentials leak, your entire AWS account is compromised
- AWS best practice: ONLY use root for initial setup and billing

âœ… USE IAM ACCOUNT WITH PROPER PERMISSIONS

Why?
- You can limit permissions (principle of least privilege)
- Better security and auditing
- Can be revoked/rotated easily
- This is the AWS recommended approach

STEP 2A: If you lack IAM permissions
-------------------------------------

Add these permissions to your IAM account:

Quick fix (full access):
    aws iam attach-user-policy \
      --user-name YOUR_IAM_USERNAME \
      --policy-arn arn:aws:iam::aws:policy/AdministratorAccess

OR manually via AWS Console:
    1. Go to: https://console.aws.amazon.com/iam/
    2. Click "Users" â†’ Find your IAM user
    3. Click "Add permissions"
    4. Attach policy: "AdministratorAccess"

================================================================================
                      WHO SHOULD ADD PERMISSIONS?
================================================================================

Option 1: If you have admin access
-----------------------------------
You can add permissions to yourself using the commands above.

Option 2: If you DON'T have admin access
-----------------------------------------
Ask your AWS account administrator to:
1. Add "AdministratorAccess" policy to your IAM user
   OR
2. Add these specific policies:
   - ElasticLoadBalancingFullAccess
   - AmazonEC2FullAccess
   - AmazonEKSClusterPolicy
   - AmazonRDSFullAccess
   - AmazonElastiCacheFullAccess
   - IAMFullAccess
   - AmazonS3FullAccess

Option 3: If you're the ONLY user/admin
----------------------------------------
1. Sign in with root account (just this once)
2. Go to IAM â†’ Users â†’ Your IAM user
3. Add "AdministratorAccess" policy
4. Sign out from root
5. Sign back in with IAM user
6. Never use root again
